# 大趋势投资雷达系统需求文档

## 1. 项目背景

用户希望建立一个系统，用于在港股、美股、A 股中识别正在形成或已经验证的中长期大趋势，避免只追逐每日题材行情。

用户的原始想法是：

> 捕捉最近半年内已经显著上涨，甚至已经翻倍的港股、美股、A 股，分析其上涨原因。如果上涨背后是 AI、算力、创新药、机器人等大逻辑，则考虑买入或纳入观察。

这个想法的核心价值是：

> 翻倍股是市场已经投票确认过的信号。

但系统不能简单理解为“股票翻倍了就买”。更合理的方式是：

> 把“半年翻倍股”作为趋势雷达的重要输入，用它反推哪些产业逻辑正在被市场重新定价，然后进一步判断这个趋势是否仍处于可参与阶段。

---

## 2. 产品目标

### 2.1 核心目标

建立一个每日/每周自动运行的投资趋势识别系统，用于：

1. 发现最近 3–6 个月已经显著走强、甚至翻倍的股票；
2. 识别这些股票背后共同的产业逻辑；
3. 判断该逻辑是短期题材、事件驱动，还是中长期大趋势；
4. 对趋势本身进行打分；
5. 在高分趋势中筛选核心个股；
6. 生成简报并推送到用户微信；
7. 帮助用户避免因为“不懂某个行业”而错过已经开始扩散的大行情。

---

## 3. 用户需求

### 3.1 用户不希望系统变成什么

用户不希望系统变成：

- 每日涨幅榜；
- 题材轮动机器人；
- 财经号式热点复盘；
- 看到什么涨就推荐什么；
- AI 主观编故事打分；
- 简单追高已经翻倍的股票。

### 3.2 用户真正想要什么

用户希望系统识别的是：

- 已经被市场验证的大趋势；
- 从龙头扩散到产业链的趋势；
- 有基本面、订单、资本开支、政策或技术周期支撑的趋势；
- 有可能持续几个月到几年的趋势；
- 在第一波上涨后，仍有第二波、第三波机会的趋势。

典型目标包括：

- AI 算力；
- 光通信 / 高速互联；
- 数据中心电力；
- 创新药出海；
- 机器人；
- 半导体设备；
- 稳定币 / 加密金融基础设施；
- 其他跨市场共振的大产业周期。

---

## 4. 对“半年翻倍股”的定位

### 4.1 保留翻倍股指标

系统必须保留用户最初提出的“半年翻倍股”思想。

半年翻倍股不是被抛弃，而是作为三类信号使用：

### A. 主线发现入口

如果某个阶段内，港股、美股、A 股中多个股票在 3–6 个月内翻倍，并且它们属于同一产业链或同一宏观逻辑，则系统应识别为潜在大趋势。

示例：

```text
多只光模块公司半年翻倍
→ 不是只看个股
→ 反推 AI 数据中心高速互联需求
→ 进一步分析 800G、1.6T、CPO、云厂商资本开支
```

### B. 趋势确认信号

如果某个主题里：

- 龙头已经翻倍；
- 二线股开始补涨；
- 上下游公司也开始创新高；
- 成交额持续放大；

则说明该主题可能已经从“局部机会”进入“主线确认阶段”。

### C. 过热 / 拥挤提醒

如果一个主题中大量股票已经半年翻倍甚至多倍，同时：

- 财经媒体密集报道；
- 估值远高于盈利上修；
- 高管或大股东减持；
- 成交额极端放大；
- 基金持仓高度拥挤；

则系统应提示：

> 该趋势可能仍然正确，但买点风险上升。

---

## 5. 系统总原则

系统遵循以下原则：

```text
价格异动先发现
产业逻辑再归因
趋势强度再打分
买点风险再判断
```

系统不直接输出：

```text
这只股票涨了很多，建议买入
```

而是输出：

```text
这条产业趋势正在变强
这些股票是趋势中的核心标的
当前处于观察、试仓、确认、过热或证伪阶段
```

---

## 6. 核心功能模块

### 6.1 市场扫描模块

覆盖市场：

- 美股；
- 港股；
- A 股。

扫描频率：

- 每日更新；
- 每周输出趋势结论；
- 每月做趋势复盘。

扫描对象：

- 全市场股票，或用户指定股票池；
- 行业 ETF；
- 主题 ETF；
- 产业链核心公司。

核心指标：

| 指标 | 用途 |
|---|---|
| 20 日涨幅 | 判断短期强度 |
| 60 日涨幅 | 判断中期趋势 |
| 120 日涨幅 | 判断半年趋势 |
| 180 日涨幅 | 判断是否接近翻倍 |
| 3–6 个月是否翻倍 | 用户原始先行 / 确认指标 |
| 距 52 周新高距离 | 判断趋势位置 |
| 成交额放大倍数 | 判断资金参与 |
| 相对大盘强度 | 判断是否跑赢市场 |
| 相对行业强度 | 判断是否真正强于同行 |
| 最大回撤 | 判断趋势健康度 |
| 波动率 | 判断是否妖股化 |

---

### 6.2 翻倍股监控模块

这是系统的独立核心模块，不能删除。

每日扫描最近 3–6 个月内：

- 涨幅超过 80% 的股票；
- 涨幅超过 100% 的股票；
- 涨幅超过 200% 的股票；
- 从低点反弹超过 100% 的股票；
- 创 52 周新高且涨幅显著的股票。

输出：

```text
最近 3 个月翻倍股列表
最近 6 个月翻倍股列表
最近 6 个月涨幅 80%–100% 的准翻倍股
最近 6 个月多倍股列表
```

但不直接推荐，而是进一步归因。

| 维度 | 说明 |
|---|---|
| 个股名称 | 股票代码、市场 |
| 涨幅 | 3 个月、6 个月、12 个月 |
| 所属行业 | 一级 / 二级行业 |
| 所属主题 | AI、创新药、机器人等 |
| 上涨原因 | 财报、订单、政策、产业趋势、事件 |
| 是否有同类股票同步上涨 | 判断是否板块共振 |
| 是否属于产业链某一环节 | 判断是否大趋势组成部分 |
| 是否有基本面验证 | 财报、订单、产能、客户 |
| 当前风险状态 | 正常、偏热、极热、证伪 |

---

### 6.3 板块 / 产业链共振模块

系统不能只看单只翻倍股，要判断是否形成趋势扩散。

判断维度：

| 维度 | 说明 |
|---|---|
| 同行业强势股数量 | 是否不止一家公司上涨 |
| 同产业链强势环节数量 | 是否从单点扩散到上下游 |
| 龙头与二线表现 | 是否从龙头扩散到二线 |
| A/H/美股映射 | 是否跨市场共振 |
| ETF/行业指数表现 | 是否有机构资金参与 |
| 成交额持续性 | 是否连续数周放量 |
| 新高股票比例 | 板块内是否大面积走强 |

示例：

```text
如果只有一只光模块公司翻倍：
可能是个股事件。

如果多只光模块、PCB、交换机、液冷、电力设备公司同步上涨：
可能是 AI 数据中心基础设施主线。

如果美股、港股、A 股都出现映射：
趋势等级进一步提高。
```

## 7. 趋势生命周期模型

系统需要判断每条趋势处于哪个阶段。

| 阶段 | 名称 | 特征 |
|---|---|---|
| 阶段 0 | 噪音 | 单日上涨，无持续性 |
| 阶段 1 | 异动 | 少数龙头开始明显走强 |
| 阶段 2 | 扩散 | 同产业链多个股票上涨 |
| 阶段 3 | 确认 | 龙头翻倍，二线补涨，成交额持续放大 |
| 阶段 4 | 业绩验证 | 财报、订单、指引开始兑现 |
| 阶段 5 | 共识强化 | 机构覆盖、盈利上修、主线被市场认可 |
| 阶段 6 | 过热 | 估值极端、舆论拥挤、交易过度 |
| 阶段 7 | 证伪 | 财报不及预期、订单下修、趋势破坏 |

用户重点想捕捉的是：

```text
阶段 2 → 阶段 5
```

不追阶段 0 的题材，也不盲目追阶段 6 的泡沫。

## 8. 打分模型设计

系统分两层打分：

```text
第一层：趋势评分
第二层：趋势内个股评分
```

这是为了避免直接在全市场里追最强股票。

---

### 8.1 趋势评分：100 分

| 模块 | 分数 | 说明 |
|---|---:|---|
| 翻倍股信号 | 15 | 该主题中是否出现多只半年翻倍 / 准翻倍股票 |
| 持续性 | 15 | 是否连续多周跑赢大盘 |
| 扩散度 | 20 | 是否从单一龙头扩散到产业链多个环节 |
| 基本面验证 | 20 | 是否有订单、收入、利润、资本开支支撑 |
| 跨市场共振 | 10 | 美股、港股、A 股是否同步映射 |
| 资金强度 | 10 | 成交额、ETF、机构资金是否持续参与 |
| 拥挤度 / 估值风险 | 5 | 是否已经过热 |
| 证伪风险 | 5 | 是否存在重大反逻辑信号 |

这里明确加入：

```text
翻倍股信号：15 分
```

也就是说，用户最开始提出的“半年翻倍股”仍然是系统的核心先行 / 确认指标之一。

---

### 8.2 翻倍股信号评分细则

| 情况 | 分数 |
|---|---:|
| 主题内多只股票 3–6 个月翻倍，且分布在多个产业链环节 | 13–15 |
| 主题内多只股票翻倍，但集中在单一环节 | 10–12 |
| 龙头翻倍，二线股涨幅 50%–80% | 7–9 |
| 只有单只个股翻倍 | 3–6 |
| 没有翻倍股，仅短线异动 | 0–2 |

---

### 8.3 趋势分层

| 趋势分 | 状态 | 系统动作 |
|---:|---|---|
| 85–100 | 已确认大趋势 | 进入核心趋势池 |
| 75–85 | 高潜力趋势 | 进入重点观察池 |
| 65–75 | 观察中 | 每周跟踪 |
| 50–65 | 可能只是轮动 | 降低权重 |
| <50 | 噪音 | 忽略 |

## 9. 个股评分模型

个股评分只在高分趋势中进行。

也就是说，系统先判断：

```text
这个趋势值不值得研究？
```

再判断：

```text
趋势里哪家公司值得跟踪？
```

### 9.1 个股评分：100 分

| 模块 | 分数 | 说明 |
|---|---:|---|
| 所属趋势强度 | 20 | 是否处于高分趋势 |
| 产业链位置 | 15 | 是否是核心环节 |
| 龙头地位 | 15 | 是否是行业龙头或高弹性标的 |
| 业绩弹性 | 15 | 收入 / 利润是否受趋势显著拉动 |
| 价格趋势 | 10 | 是否处于健康上升趋势 |
| 估值可解释性 | 10 | 当前估值能否被增速解释 |
| 流动性 | 5 | 成交额是否足够 |
| 风险项 | 10 | 减持、客户集中、财务风险、暴涨过热等 |

## 10. AI 的角色

AI 不负责主观拍脑袋打分。

AI 负责：

1. 阅读新闻；
2. 阅读公告；
3. 阅读财报；
4. 归纳上涨原因；
5. 提取产业逻辑；
6. 判断证据类型；
7. 识别风险；
8. 生成结构化标签。

AI 输出格式示例：

```json
{
  "股票": "某光通信公司",
  "上涨原因": [
    "AI 数据中心需求增长",
    "800G/1.6T 光模块订单提升",
    "海外云厂商资本开支持续上修"
  ],
  "证据类型": [
    "财报",
    "订单",
    "行业资本开支"
  ],
  "逻辑级别": "产业级",
  "可持续性": "中高",
  "主要风险": [
    "估值较高",
    "客户集中",
    "技术迭代风险"
  ],
  "是否属于大趋势": true
}
```

然后系统根据标签映射到规则分数。

## 11. 每日 / 每周简报需求

### 11.1 推送频率

建议分两类。

#### 每日简报

每日更新，但内容克制，不追短线热点。

内容包括：

1. 趋势分变化最大的主线；
2. 新进入高潜力区的趋势；
3. 新出现的半年翻倍 / 准翻倍股；
4. 是否有产业链扩散信号；
5. 是否有趋势证伪或过热信号。

#### 每周完整版

每周末输出更完整的趋势报告。

内容包括：

1. 趋势排行榜；
2. 最近 3–6 个月翻倍股分布；
3. 新兴主线分析；
4. 产业链扩散图；
5. 核心股票池变化；
6. 下周关键验证点；
7. 风险提醒。

---

```markdown
# 大趋势雷达日报
日期：2026-XX-XX

## 一、趋势仪表盘

1. AI 光通信；

3. 创新药出海。

系统判断：
这些不是单只个股异动，已经出现产业链扩散迹象。

## 三、新进入观察池的趋势

### 数据中心电力

趋势分：79/100
状态：高潜力趋势

触发原因：
- 多只相关股票 3–6 个月涨幅显著；
- AI 数据中心建设带动电力需求；
- 美股、A 股均出现相关映射；
- 成交额连续数周放大。

需要验证：
- 云厂商资本开支是否继续上修；
- 相关公司订单是否进入财报；
- 估值是否已经过热。

## 四、今日新增翻倍股 / 准翻倍股

## 五、系统结论

今日没有短线交易建议。

1. AI 算力基础设施是否继续扩散；
2. 创新药出海是否出现更多订单 / BD 验证；
3. 数据中心电力是否从海外映射到 A/H 股。
```

不建议使用个人微信机器人。

推荐方式：

| 方式 | 推荐度 | 说明 |
|---|---:|---|

MVP 阶段推荐：

```text
PushPlus 或 Server 酱
```

## 14. MVP 版本范围

### 14.1 第一版不要太复杂

MVP 目标：

```text
先验证系统能不能比人更早发现主线扩散。
```

第一版只做：

1. 港股、美股、A 股核心股票池；
2. 最近 20 / 60 / 120 / 180 日涨幅；
3. 半年翻倍 / 准翻倍股监控；
4. 行业 / 主题聚类；
5. 趋势评分；
6. AI 归因；
7. 每日简报；
8. 微信推送。

### 14.2 MVP 暂时不做

## 15. 系统判断逻辑总结

系统不是：

```text
股票翻倍 → 买
```

而是：

```text
股票翻倍
→ 归因上涨原因
→ 找同类股票
→ 判断是否板块共振
→ 判断是否产业链扩散
→ 判断是否有基本面验证
→ 判断趋势阶段
→ 判断是否过热
→ 输出趋势评分和观察池
```

---

## 16. 最重要的产品设计修正

之前讨论中，大趋势识别容易看起来像是弱化了“翻倍股”这个先行指标。

更准确的版本应该是：

> 翻倍股不是最终买入信号，但它是系统发现大趋势的核心入口之一。

具体定位如下：

| 翻倍股的作用 | 是否保留 |
|---|---|
| 发现市场正在奖励什么方向 | 保留 |
| 反推产业链主线 | 保留 |
| 判断趋势是否被市场确认 | 保留 |
| 作为趋势评分的一部分 | 保留 |
| 直接作为买入依据 | 不采用 |
| 用来判断过热风险 | 保留 |

最终系统应该是：

```text
翻倍股扫描系统
+ 趋势扩散识别系统
+ 基本面归因系统
+ 趋势生命周期判断系统
+ 微信简报系统
```

而不是简单的：

```text
每日题材雷达
```

---

## 17. 一句话产品定义

这是一个以“半年翻倍股”为重要入口，结合产业链扩散、跨市场共振、基本面验证和趋势生命周期判断的大趋势投资雷达系统。

它的目的不是追每日热点，而是帮助用户识别：

> 哪些产业逻辑正在被市场持续重新定价，并且仍可能处于可参与阶段。
