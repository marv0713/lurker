# 职业资金雷达实现说明

## 范围

当前职业资金雷达只服务 A 股。港股、美股、ETF 申赎、股指期货贴水、期权 IV 暂不接入。

默认日报策略为 `professional_flow_daily`，旧的 `long_term_trend` 保留为 legacy 策略。

## 数据快照

资金流快照保存到：

```text
data/processed/flow_snapshots/YYYY-MM-DD.json
```

核心字段：

- `market_flow`：大盘主力、超大单、大单资金流。
- `sector_flows`：行业/板块资金流排名。
- `stock_flows`：个股今日、5日、10日资金流合并结果。
- `margin`：两融余额及与上一份缓存的余额变化。
- `failures`：采集失败和降级原因。

## 已修复的数据口径问题

### 1. 个股资金流必须合并多周期

AkShare 的 `stock_individual_fund_flow_rank(indicator="今日")` 只返回今日资金流，不包含 5日、10日字段。

现在 `fetch_stock_flows()` 会分别抓取：

```text
今日
5日
10日
```

然后按股票代码合并。否则个股资金分最多只能到 50 分，永远达不到 `2%候选` 的资金门槛。

### 2. 大盘资金流取最新交易日

`stock_market_fund_flow()` 返回历史序列。归一化时会按 `日期` 排序并取最新一行。

如果没有 `日期` 字段，则使用最后一行。

### 3. 两融变化来自缓存差值

两融接口返回余额，不直接返回本系统需要的 `margin_balance_change`。

现在 `fetch_margin()` 会读取上一份 `data/processed/margin_cache.json`，用当前 `margin_balance` 减去上一份余额，生成：

```text
margin_balance_change
```

首次运行或无缓存时不生成变化值。市场温度不会把缺失的两融变化当作正向确认。

### 4. AkShare 网络处理不再全局改写 requests

早期实现会在 `flows.py` import 时 monkeypatch `requests.get/post`，这会影响 PushPlus、其他 HTTP 代码和测试进程。

现在只在 AkShare fetcher 调用期间临时 patch，并在调用结束后恢复。

### 5. 推送校验不再要求个股资金流非空

职业资金日报可能只有市场/板块资金，没有个股资金候选。这样的报告仍然有价值，允许推送。

只有当价格快照为空，或资金快照完全没有 `market_flow`、`sector_flows`、`stock_flows` 时，才跳过推送。

## 市场温度规则

`进攻` 需要：

- 大盘主力资金净流入。
- 超大单净流入。
- 两融变化非负，或核心 ETF 明显放量。

`防守` 需要：

- 大盘主力资金净流出。
- 超大单净流出。
- 核心 ETF 未放量。

其他情况为 `观察`。

## 当前限制

- 个股资金流只取 AkShare 排名接口中出现的股票，接口覆盖不等于全市场完整明细。
- 板块与主题映射仍有人工规则，例如 `ai_infra`、`innovative_drugs` 到东方财富行业名的映射。
- 两融变化依赖本地缓存；首次运行只提供余额，不提供变化。
- 核心 ETF 放量字段目前依赖外部传入 `core_etfs`，还没有独立采集实现。

## 弹簧买点观察口径

`弹簧买点观察` 不是所有未进入 `2%候选` 的股票集合。

只有同时满足以下条件，才会进入该区：

- 综合分数不为负。
- `setup_score >= 60`。
- 没有方向性背离，例如 `强势未获资金确认`、`资金试探`、`跟风不足`。
- 市场温度不是 `防守`。

负分标的、背离标的、趋势破坏标的应进入 `证伪/退潮提醒` 或不展示，不能放进弹簧买点观察。

## 周报口径

命令：

```text
PYTHONPATH=src .venv/bin/lurker weekly-report --date YYYY-MM-DD
```

周报聚合 `data/processed/flow_snapshots/` 中报告日期之前最近 N 份可用交易日快照，默认 N=5。这里的 `--lookback` 不是自然日窗口，而是“最近 N 个快照”，避免周末或节假日导致周报漏掉周初交易日。

周报重点回答三件事：

- 本周市场状态：按每日 `进攻 / 观察 / 防守` 统计。
- 本周资金主线：按板块正流入天数、累计主力净流入、最新排名排序，默认只展示前 10 个。
- 核心股票资金流向：按个股正流入天数和累计净流入排序，默认只展示前 20 个。

周报会保留“本周曾经强、最新一天转负”的板块，并标记为 `退潮`。这类主线不能因为周五转负就消失，否则复盘时看不到资金从哪里撤退。

过滤规则：

- 板块只纳入至少 1 天主力净流入为正的对象。
- 个股过滤 `ST`、`*ST`、`退市` 名称。
- 默认数量可通过 `--sector-limit`、`--stock-limit` 调整，防止推送内容膨胀。

`main_candidates_count` 记录的是周报展示的主线板块数量，不是快照数量。
