# A-Share Macro Flow Radar — 周报与月报设计

Date: 2026-06-04

## Goal

在日报（`professional_flow_daily`）之外，增加两个更高维度的宏观资金视角：

- **周报**（`weekly_macro_flow`）：每周五收盘后运行，回答"这一周资金是不是在持续进场"。
- **月报**（`monthly_macro_flow`）：每月最后一个交易日运行，回答"当前是否处于牛市进度条上升阶段"。

日报侧重个股和板块的当日动向；周报和月报侧重宏观流动性趋势，用来判断日报信号的大背景。

---

## 周报：`weekly_macro_flow`

### 报告标题

`宏观资金周报`

### 回答的问题

- 这一周资金是不是持续在进场？
- 哪些板块连续跑赢？哪些只是反弹？
- 超大单周度净流入是否比价格表现更强？
- ETF 是净申购还是净赎回？

### 数据源

#### ETF 净申购趋势

- 核心 ETF（510300、510500、159915 以及可配置的 A500 ETF）的周度申购/赎回变化。
- 使用 AkShare `fund_etf_hist_em` 获取 ETF 成交量和份额变化作为申购强度代理。
- 信号：
  - 连续两周净申购 → `持续进场`
  - 本周净赎回 → `资金撤退`
  - 份额平稳但价格上涨 → `存量博弈`

#### 两融余额周度变化

- 使用 Tushare `margin`（需 `TUSHARE_TOKEN`），按周汇总。
- 四档判断：

| 状态 | 条件 | 含义 |
|------|------|------|
| `健康上升` | 周度增速 1–5% | 杠杆温和扩张 |
| `过热预警` | 周度增速 > 5% | 杠杆扩张过快 |
| `高位盘整` | 绝对值高位但增速 < 1% | 警惕转折 |
| `恐慌出清` | 周度降幅 > 3% | 可能是底部信号 |

#### 超大单周度净流入

- 使用 AkShare `stock_market_fund_flow` 汇总一周的超大单净流入。
- 对比当周指数涨跌：
  - 指数平但超大单净流入 → `聪明钱进场，价格滞后`
  - 指数涨但超大单净流出 → `散户拉盘，主力减仓`
  - 两者同向上涨 → `共振进攻`

#### 板块强弱周度排序

- 使用 AkShare `stock_sector_fund_flow_rank` 获取行业和概念板块的周度资金流。
- 连续性判断（对比上周排名）：
  - 连续两周前5 → `持续主线`
  - 本周进入前5、上周不在 → `新兴热点`
  - 上周前5、本周跌出 → `热点退潮`
  - 资金流入但价格跑输大盘 → `资金滞留，吸筹`

### 报告格式

```text
# 宏观资金周报

周期：YYYY-MM-DD 至 YYYY-MM-DD

## 一句话结论

## ETF 资金动向

## 两融余额状态

## 超大单周度信号

## 板块强弱排序

## 数据质量
```

### 评分与结论

综合以上四项输出一个周度市场状态：

- `积极进场`：ETF 净申购 + 超大单净流入 + 两融健康 + 板块主线明确
- `谨慎观望`：信号混杂，至少一项出现矛盾
- `风险上升`：两融过热或 ETF 持续净赎回
- `恐慌区间`：两融快速下降 + ETF 净赎回 + 超大单净流出

## 月报：`monthly_macro_flow`

### 报告标题

`宏观流动性月报`

### 回答的问题

- 现在是不是牛市进度条在往前走？
- 存款搬家是否在加速？
- 杠杆水位是否健康？

### 数据源

#### 居民存款增速

- 数据来源：中国人民银行月度数据，需通过 AkShare `macro_china_money_supply` 或 Tushare `cn_m2` 获取。
- 枯荣线：同比增速 **12%**。
  - 低于 12% → 存款增速放缓，资金有搬家迹象
  - 高于 12% → 居民资产仍以存款为主，牛市还早

#### 非银存款变化

- 非银存款上升 = 资金进入金融市场（券商、基金等）。
- 赚钱效应扩散的信号之一。
- 数据：AkShare `macro_china_money_supply` 中的 M1/M2 差值可作为代理（M1 上升快于 M2 通常意味着活钱增多）。

#### 融资余额占流通市值

- 融资余额：Tushare `margin` 汇总。
- 流通市值：AkShare `stock_market_activity_legu` 或固定用沪深市场总流通市值估算。
- 两条警戒线：
  - 比例 > **4%** → 危险区
  - 月度增速 > **20%** → 杠杆过热

#### M1-M2 剪刀差（辅助）

- M1 增速 - M2 增速。
- 持续扩大 → 经济活跃度上升，有利于股市。
- 持续收窄或负值 → 活钱减少，市场可能低迷。

### 报告格式

```text
# 宏观流动性月报

月份：YYYY-MM

## 一句话结论

## 牛市进度条

## 居民存款趋势

## 非银存款与活钱指标

## 杠杆水位

## 数据质量
```

### 月度市场状态分级

- `牛市加速`：存款增速破枯荣线 + 非银存款上升 + 杠杆健康
- `慢牛蓄力`：部分指标改善，方向对但幅度不足
- `震荡磨底`：指标混杂，无明确方向
- `过热警报`：杠杆超 4% 或增速超 20%

## 数据源可用性评估

| 指标 | 数据源 | 依赖 | 可用性 |
|------|--------|------|--------|
| ETF 份额/成交量 | AkShare `fund_etf_hist_em` | 无 | ✅ 无需配置 |

| 超大单周度流入 | AkShare `stock_market_fund_flow` | 无 | ✅ 无需配置 |

| 居民存款 | AkShare 宏观数据 | 无 | ⚠️ 需测试可用性 |
| 非银存款 | AkShare 宏观数据 | 无 | ⚠️ 需测试可用性 |
| 融资余额/流通市值 | Tushare + 估算 | `TUSHARE_TOKEN` | ✅ 可计算 |

## Rollout

### Phase 1（周报）
- 新增 `src/lurker/ingest/macro_flows.py`：抓取 ETF 申购、板块周度资金流、超大单汇总、两融周度数据。
- 新增 `src/lurker/application/weekly_macro_flow.py`：周度评分与报告生成。
- 在 `configs/strategies.yaml` 中启用 `weekly_macro_flow`，cadence=weekly。
- 新增 `tests/test_weekly_macro_flow.py`。

### Phase 2（月报）
- 新增 `src/lurker/ingest/macro_monthly.py`：拉取 M1/M2、居民存款、融资余额。
- 新增 `src/lurker/application/monthly_macro_flow.py`：月度打分与报告。
- 在 `configs/strategies.yaml` 中启用 `monthly_macro_flow`，cadence=monthly。

### Phase 3（整合）
- 在日报"市场资金温度"章节引用上一期周报和月报的宏观结论，形成三层嵌套视角。

---

## Non-Goals

- 不做个股级别的周度/月度分析（个股用日报即可）。
- 不预测价格，只描述资金流向和市场状态。
- 不要求实时数据，允许 T+1 延迟。
