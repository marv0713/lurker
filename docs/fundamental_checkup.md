# 自选股基本面异常体检 (Fundamental Anomaly Checkup) 开发文档

## 一、 模块定位
本模块旨在为长线投资者提供**极低频、高信噪比**的持仓监控服务。
它独立于全市场长线趋势扫描模块，仅针对用户填写的 `configs/watchlist.yaml` 个人观察池生效。平时保持静默，仅当标的发生足以动摇长线基本面逻辑的“极端异动”时触发警报。

## 二、 核心警报指标算法定义

### 1. 巨量暴跌/暴涨报警 (Abnormal Volume & Price Shift)
**场景**：财报超预期/暴雷、被做空、重大行业政策发布。长线股最怕不知不觉被边缘化，巨量异动代表大资金正在激烈博弈。
*   **计算公式**：
    *   `volatility_ratio = 当前日成交量 / 过去20个交易日平均成交量`
    *   `price_change = abs((当前收盘价 - 昨日收盘价) / 昨日收盘价)`
*   **触发阈值**：
    *   `volatility_ratio >= 3.0` (即放量 3 倍以上)
    *   `price_change >= 0.05` (A股/港股涨跌幅 >= 5%) 或 `0.10` (美股涨跌幅 >= 10%，美股无涨跌幅限制因此阈值更高)
*   **警报文案模板**：`🚨 巨量异动：[代码] 今日放量 [X] 倍，涨跌幅 [Y]%。大资金正在激烈博弈，请密切关注近期公告/新闻！`

### 2. 高位重度回撤报警 (Significant Drawdown from Peak)
**场景**：长牛股的正常回调一般在 10% 以内，若回撤超过 20%，通常意味着趋势逆转，长线成长逻辑可能被破坏（进入技术性熊市）。
*   **计算公式**：
    *   拉取过去 250 个交易日（约一年）的最高价 `High_250`。
    *   `drawdown = (当前收盘价 - High_250) / High_250`
*   **触发阈值**：
    *   `drawdown <= -0.20` (回撤幅度超过 20%)
*   **警报文案模板**：`⚠️ 高位回撤：[代码] 已从一年内最高点回撤 [X]%，进入熊市区间。请审视其长牛基本面逻辑是否恶化！`

### 3. 持续跑输大盘报警 (Chronic Underperformance)
**场景**：持有成长股的初衷是为了超额收益（Alpha）。若长线个股持续数月跑输基准指数，说明资金正在持续流出。
*   **计算公式**：
    *   获取标的过去 60 个交易日（约 3 个月）的累计涨跌幅 `Return_60d`。
    *   获取该标的对应市场的基准指数过去 60 日涨跌幅 `Benchmark_60d`（美股取 SPY，A股取 沪深300，港股取 恒生指数）。
    *   `alpha_60d = Return_60d - Benchmark_60d`
*   **触发阈值**：
    *   `alpha_60d <= -0.15` (跑输基准指数 15% 以上)
*   **警报文案模板**：`📉 持续跑输：[代码] 近 3 个月大幅落后大盘 [X]%。成长溢价正在消退，请评估资金是否已转移。`

## 三、 工程实现路径 (Implementation Steps)

1. **依赖配置文件建立**
   * 创建 `configs/watchlist.yaml` 读取用户的长线自选池。
2. **数据源 (Ingest) 扩容**
   * 改造 `src/lurker/ingest/prices.py`，针对自选池中的股票，除了原有的收盘价外，必须抓取：
     - `Volume`（当前成交量）
     - 过去 20 日的历史 `Volume`
     - 过去 250 日的 `High` (历史最高价)
     - 对应市场的基准指数 (Benchmark) 行情数据
3. **分析引擎 (Signals)**
   * 在 `src/lurker/signals/anomaly.py` 中新增这 3 种警报的探测类。
4. **格式化输出 (Reports)**
   * 在 `daily_report.py` 内部单独劈出一个 `render_watchlist_alerts()` 的模块。
   * 当检测结果列表为空时，不在日报中显示该版块。

## 四、 扩展性设计
该模块在代码设计上需遵循 `Strategy` 模式，确保未来用户随时可以开启/关闭特定警报（如关闭“跑输大盘报警”），或自行调节参数（如将放量 3 倍改为放量 2 倍）。
