# 投资雷达系统架构与设计理念演进记录

本文档用于记录项目在开发过程中，基于投资逻辑和实际需求发生的重大设计理念演进与讨论记录。

## 2026-05-30：从“短期板块轮动”向“长期成长股与异常体检”的全面转向

### 1. 痛点与需求变更
*   **初始设想**：通过抓取全市场板块、概念主题的 20/60/180 日动量，帮助投资者无偏见地发现短期热点（如 AI、光模块）并进行板块切换。
*   **实际痛点**：用户的投资风格偏向**“长期观察、灵活持有成长股，预期持有一年以上”**。短期板块轮动的思路（打地鼠）并不适合该风格。用户的核心痛点在于：
    1.  个人的股票池太小，由于不炒短线，容易错过真正走出一两年长牛的新标的（红利错失）。
    2.  平时不看技术线（如均线），且没时间频繁盯盘。对于长期持仓的股票，如果基本面或中期趋势发生重大恶化，容易后知后觉。

### 2. 架构理念重构：“双引擎”系统
基于上述痛点，雷达系统废弃了传统的短期动量打分逻辑，重构为以下双引擎：

#### 引擎一：长牛成长股发现器 (Long-Term Discovery Engine)
*   **目标**：无偏见地扫描全市场，筛选出处于“长牛通道”的新候选标的，扩充用户的长线股票池。
*   **逻辑**：
    *   **观察池基础**：只保留代表“成长与创新”的宽基和行业 ETF（摒弃防御性和夕阳周期板块）。
    *   **过滤指标**：放弃 20 日短线指标，核心看重 **180 日（半年）** 和 **250 日（一年）** 的动量。
    *   **趋势确认**：引入 **“创 52 周新高距离”** 指标，只筛选贴近历史高点、拒绝深幅回撤死猫反弹的强势标的。

#### 引擎二：自选股“异常体检”私人助理 (Watchlist Anomaly Alerts)
*   **目标**：针对用户长期持有的自选股，平时保持绝对安静（不打扰），仅在发生足以影响长线逻辑的“极端异常”时发出警报。
*   **逻辑**（不看技术线，只看极端基本面/资金面异动）：
    1.  **巨量暴跌/暴涨报警 (Abnormal Volume)**：单日成交量激增（如 > 3倍），且伴随剧烈价格波动（如 > 5%），提示可能有财报雷、做空报告或重大政策变化，需人工查阅新闻。
    2.  **高位重度回撤报警 (Peak Drawdown)**：从 52 周最高点回撤超过 20%，提示该成长股可能进入中期技术性熊市，成长溢价逻辑可能被破坏。
    3.  **持续跑输大盘报警 (Chronic Underperformance)**：连续 3 个月涨幅严重落后于基准大盘，提示资金可能已发生长线转移。

### 3. 关于引入海外社交媒体 (Twitter/Reddit) 情绪的讨论
*   **结论**：放弃引入。
*   **原因**：社交媒体情绪（如 WallStreetBets）充满噪音且滞后。真正的大趋势会在早期体现在价格动量和成交量的异动上（聪明钱先行）。此外，社交媒体的 API 成本和 NLP 处理复杂度假高，ROI 极低。坚持“价格包容一切”的量化扫描是最客观的方式。
